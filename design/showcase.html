Design System Overview

A comprehensive design system for creating Ethereum-themed trading cards with cyberpunk aesthetics and high-finance elements. Built with composable SVG templates for rapid card creation while maintaining visual consistency.

Composable Templates

Layered SVG templates with separate elements for background, frame, text, art, and icons allow for rapid customization and consistent styling across all card types.

Cyberpunk Aesthetics

Futuristic design language combining blockchain iconography with high-finance themes. Dark backgrounds with neon accents create dramatic visual impact.

Type-Based Hierarchy

Each card type uses distinct color schemes and visual motifs for instant recognition: Chain (blue), DeFi (green/orange), Action (red), EOA (cyan/gold), Ability (gray/purple).

Scalable System

Vector-based templates support both digital and print formats. Modular icon system allows for easy expansion as new mechanics are added to the game.

Template System

Five base templates corresponding to each card type, designed for easy customization while maintaining visual consistency.

Technical Specifications

Card Dimensions

300x420 pixels (standard TCG proportions)
Art area: 270x180 pixels
Safe margins: 15px from edges

File Format

SVG vector format for scalability
RGB color space for digital
CMYK conversion for print

Typography

Minimum 12px text size
Arial/sans-serif font family
High contrast for readability

Print Requirements

300 DPI minimum resolution
3mm bleed for print production
Rounded corners: 15px radius

Template Features

Placeholder System

Each template uses standardized placeholders like {CARD_NAME}, {COST}, {DESCRIPTION} for easy batch production and automation.

Layered Structure

Separate layers for background, frame, text, art, and icons enable non-destructive editing and component reuse.

Responsive Layout

Elements scale proportionally and maintain legibility across different output sizes from thumbnails to print.

Variant Support

Templates accommodate card variants within types, such as different staking mechanics or ability effects.

Quality Standards

Visual Consistency
  • Color schemes match card type
  • Typography follows hierarchy
  • Icon usage is consistent
  • Rarity indicates power level
Technical Accuracy
  • All placeholders replaced
  • Text is legible at card size
  • SVG code is optimized
  • Files properly named
Game Integration
  • Abilities match game mechanics
  • Cost balancing is appropriate
  • Art fits established lore
  • Effects are clearly communicated
